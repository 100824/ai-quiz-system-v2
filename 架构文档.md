# 小学AI课堂问卷调查系统 架构文档

## 项目概述

这是一个面向小学AI课堂的问卷调查系统，分为学生答题端和教师管理后台，支持12个课程、多个班级的问卷调查，问卷分为四个部分，教师可以实时控制进度、查看统计数据并导出Excel。

**项目名称：** ai-quiz-system-v2  
**版本：** 2.0.0  
**技术栈：** Node.js + Express + SQLite + 原生HTML/CSS/JavaScript

---

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     客户端浏览器                               │
│  ┌──────────────┐  ┌──────────────┐                          │
│  │  学生端      │  │  教师端      │  纯静态HTML+JS，通过AJAX  │
│  │ student.html │  │ teacher.html │  调用后端API             │
│  └──────────────┘  └──────────────┘                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP RESTful API
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   后端服务 Node.js + Express                 │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐ │
│  │ 接口路由   │ │ 日志模块   │ │ 数据库操作 │ │ Excel导出  │ │
│  │ 学生端API  │ │ winston    │ │ better-sqlite3          │ │
│  │ 教师端API  │ │ 访问日志   │ │ WAL模式高并发            │ │
│  └────────────┘ └────────────┘ └────────────┘ └────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ SQL
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   数据存储 SQLite 数据库                       │
│  data/quiz-system.db                                          │
│  - 7张表存储所有业务数据                                      │
│  - WAL模式支持并发读写                                        │
└─────────────────────────────────────────────────────────────┘
```

### 架构特点

1. **前后端一体化单进程部署**：前端静态文件由后端直接托管，不需要单独部署前端
2. **全栈JavaScript**：前后端都使用JavaScript，技术栈统一，开发维护简单
3. **轻量级**：使用SQLite作为数据库，不需要额外安装MySQL等数据库服务
4. **云原生友好**：可以使用PM2进行进程管理，支持守护进程和日志管理

## 后端架构

### 技术栈

- **Web框架：** Express 4.18.2
- **数据库：** better-sqlite3 9.4.3（同步API，性能更好）
- **CORS支持：** cors 2.8.5
- **请求解析：** body-parser 1.20.2
- **Excel导出：** xlsx 0.18.5
- **日志：** winston 3.11.0
- **进程管理：** PM2 5.3.1（生产环境推荐）

### 入口文件：`backend/server.js`

#### 主要中间件

| 中间件 | 作用 |
|--------|------|
| `cors()` | 支持跨域请求 |
| `bodyParser.json()` | 解析JSON请求体，支持最大10MB |
| 日志中间件 | 记录每个请求的方法、路径、状态码、处理时间 |
| `express.static()` | 托管前端静态文件 |

#### API 接口分类

**1. 教师端接口** (`/api/teacher/*`)

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/teacher/courses` | GET | 获取所有课程列表和当前课程ID |
| `/api/teacher/current-course` | POST | 切换当前课程 |
| `/api/teacher/class-list/:courseId/:className` | GET | 获取班级学生名单 |
| `/api/teacher/class-list` | POST | 导入班级名单 |
| `/api/teacher/classes/:courseId` | GET | 获取课程下所有班级 |
| `/api/teacher/stage/:courseId` | GET | 获取课程当前阶段 |
| `/api/teacher/stage` | POST | 设置课程/班级当前阶段 |
| `/api/teacher/bind-class-course` | POST | 绑定班级和课程 |
| `/api/teacher/tip` | POST | 提交课堂提示语 |
| `/api/teacher/questions/:courseId/:part` | GET | 获取某一部分的题目列表 |
| `/api/teacher/question/:id` | POST | 更新题目 |
| `/api/teacher/part2-guide/:courseId` | GET | 获取第二部分答题指引 |
| `/api/teacher/part2-guide/:courseId` | POST | 更新第二部分答题指引 |
| `/api/teacher/stats/:courseId` | GET | 获取统计数据（各部分完成情况、得分分布、正确率等） |
| `/api/teacher/stats/:courseId/export` | GET | 导出统计数据为Excel |

**2. 学生端接口** (`/api/student/*`)

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/student/validate` | POST | 验证学生（检查是否在班级名单中） |
| `/api/student/tip` | GET | 获取课堂提示语 |
| `/api/student/:studentId` | GET | 获取学生问卷状态和历史分数 |
| `/api/student/questions/:part` | GET | 获取指定部分的题目 |
| `/api/student/:studentId/part1` | POST | 提交第一部分答案 |
| `/api/student/:studentId/part2` | POST | 提交第二部分答案 |
| `/api/student/:studentId/part3` | POST | 提交第三部分答案并计算得分 |
| `/api/student/:studentId/part4` | POST | 提交第四部分答案，完成问卷 |

### 数据库设计：`backend/database.js`

使用 better-sqlite3 同步API，性能更好，代码更简洁。数据库开启WAL模式，支持高并发读写。

#### 表结构设计

| 表名 | 作用 | 主要字段 |
|------|------|----------|
| `system_config` | 系统配置 | `key`, `value` - 存储当前课程ID等配置 |
| `courses` | 课程表 | `id`, `name`, `lesson_number`(1-12), `description`, `part2_guide` |
| `class_lists` | 班级名单 | `course_id`, `class_name`, `student_name` - 每个课程下的班级学生 |
| `course_stages` | 课程阶段表 | `course_id`, `class_name`, `current_stage` - 每个班级当前进度 |
| `class_course_bind` | 班级课程绑定 | `class_name`, `course_id` - 每个班级绑定对应课程 |
| `questions` | 题目表 | `course_id`, `part`(1-4), `question_type`, `question_text`, `options`(JSON), `correct_answer`, `explanation` |
| `student_surveys` | 学生问卷表 | `course_id`, `student_id`(班级_姓名), `student_name`, `class_name`, `part1_answers`(JSON), `part2_answer`, `part3_answers`(JSON), `part3_score`, `part4_answers`(JSON), `submitted_at` |

#### 关键设计点

1. **多课程支持**：支持12节课，每节课可以自定义题目
2. **按班级控制进度**：不同班级可以在不同阶段，支持独立进度控制
3. **JSON存储灵活数据**：题目选项和学生答案使用JSON格式存储，适应不同题目类型
4. **WAL模式**：SQLite开启WAL模式，支持并发读取，写入不阻塞读取

### 日志模块：`backend/logger.js`

使用 winston 记录日志，输出到控制台和文件，支持不同日志级别。

## 前端架构

### 技术栈

- **纯原生开发**：不使用框架，直接HTML + CSS + JavaScript
- **响应式设计**：适配手机和平板屏幕，学生可以在平板上直接答题
- **无构建步骤**：修改代码直接刷新浏览器即可生效，开发简单

### 入口页面

| 页面 | 入口 | 功能 |
|------|------|------|
| 学生端 | `/student.html` | 学生登录、分步答题 |
| 教师端 | `/teacher.html` | 教师管理课程、班级、题目、查看统计、导出数据 |

### 前端目录

```
frontend/
├── css/
│   └── style.css          # 全局样式，响应式布局
├── js/
│   ├── student.js         # 学生端：登录验证、分步加载题目、提交答案
│   └── teacher.js         # 教师端：课程管理、班级管理、题目编辑、统计展示
├── student.html
└── teacher.html
```

### 学生端主要流程

```
登录页面 → 选择班级 → 输入姓名 → 验证 → 加载题目
   ↓
按阶段显示：第一部分 → 提交 → 等待教师开启第二部分
   ↓
第二部分 → 提交 → 等待教师开启第三部分
   ↓
第三部分 → 提交 → 立即显示得分和解析 → 等待教师开启第四部分
   ↓
第四部分 → 提交 → 完成
```

### 教师端主要功能

1. **课程管理**：切换课程、控制班级进度、发布课堂提示语
2. **班级管理**：导入班级名单（粘贴姓名列表，一行一个）
3. **题目管理**：编辑每个部分的题目，支持单选、多选、判断、文本题
4. **统计数据**：查看各部分完成情况、得分分布、每题正确率，导出Excel

### 教师端安全

- 进入教师端需要输入4位数字密码（密码在 `teacher.js` 中配置）
- 没有用户系统，简单密码防护足够

## 问卷四个部分设计

系统将问卷分为四个部分，教师逐步开启：

| 部分 | 名称 | 内容 | 说明 |
|------|------|------|------|
| 第一部分 | 上课前 | 预测得分、学习方法选择 | 课前填写 |
| 第二部分 | 思考与讨论 | 开放题，关于AI的思考 | 课堂讨论后填写 |
| 第三部分 | 小测验 | 5道选择题测验，即时出分 | 课后填写 |
| 第四部分 | 学习后 | 对比实际得分和预测得分、反思学习方法 | 全部完成后填写 |

## 部署架构

### 本地开发

```bash
npm install
npm run dev  # 使用nodemon开发，自动重启
```

服务启动在 `http://0.0.0.0:8080`

- 学生端：`http://服务器IP:8080/student.html`
- 教师端：`http://服务器IP:8080/teacher.html`

### 生产环境部署（使用PM2）

```bash
# 启动
npm run pm2:start

# 查看日志
npm run pm2:logs

# 停止
npm run pm2:stop
```

PM2配置文件：`ecosystem.config.js`

```javascript
module.exports = {
  apps: [{
    name: 'ai-quiz-system',
    script: 'backend/server.js',
    watch: false,
    instances: 1,
    autorestart: true,
    max_memory_restart: '1G',
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
}
```

## 数据流向

### 学生答题流程

```
学生在浏览器输入姓名班级
         ↓
前端调用 POST /api/student/validate 验证
         ↓
后端查询 class_lists 表验证
         ↓
验证通过，前端调用 GET /api/student/questions/:part 获取题目
         ↓
学生作答，点击提交
         ↓
前端调用 POST /api/student/:studentId/partX 提交答案
         ↓
后端保存到 student_surveys 表
         ↓
第三部分提交时后端计算得分，返回每题解析
         ↓
前端展示结果
```

### 教师导出Excel流程

```
教师点击导出按钮
         ↓
前端调用 GET /api/teacher/stats/:courseId/export
         ↓
后端查询 student_surveys 表所有数据
         ↓
后端拆分JSON字段，整理成表格数据
         ↓
使用xlsx生成Excel文件
         ↓
后端返回文件流，浏览器下载
```

## 主要改进点（v2 对比 v1）

1. **按班级独立进度**：不同班级可以在不同阶段，支持多个班级同时上不同课程
2. **班级课程绑定**：每个班级绑定一个课程，学生登录自动选择正确课程
3. **第二部分答题指引**：教师可以自定义第二部分答题指引，支持富文本
4. **学生历史分数对比**：学生可以看到自己之前课程的预测得分和实际得分对比
5. **Excel导出拆分字段**：导出时将JSON字段拆分为多列，方便数据分析
6. **学生姓名自动匹配**：输入1个字符即可自动匹配姓名，避免输入错误
7. **数据库WAL模式**：提高并发性能和数据安全性
